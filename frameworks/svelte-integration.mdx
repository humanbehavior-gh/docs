### Option 1: Installation Wizard (Recommended)

The easiest way to get started is using our installation wizard:

```bash
# Navigate to your project directory
cd your-project-directory

# Run the installation wizard
npx humanbehavior-js YOUR_API_KEY
```

The wizard will:
1. Ask for your API key (if not provided as an argument)
2. Let you select **Svelte** from the framework options
3. Automatically install and configure the SDK for your Svelte project:
   - Install the `humanbehavior-js` package
   - Add humanBehaviorStore to your main file
   - Configure environment variables
   - Set up proper Svelte integration

**Version Compatibility**: The wizard automatically detects your Svelte setup:
- **SvelteKit**: Detects `svelte.config.js`
- **Regular Svelte**: Standalone Svelte projects
- TypeScript support (`typescript` or `svelte-check`)
- Router usage (`svelte-routing` or `@sveltejs/kit`)
- Package managers: npm, yarn, pnpm

### Option 2: Manual Setup

If you prefer manual setup or the wizard doesn't work for your project, follow the steps below.

## Step 1: Installation

```bash
npm install humanbehavior-js
# or
yarn add humanbehavior-js
```

## Step 2: Environment Variables

Create an environment file (`.env`) with your API key:

```bash
PUBLIC_HUMANBEHAVIOR_API_KEY=your-api-key-here
```

## Step 3: Basic Setup

### For SvelteKit (SSR-enabled)

Update your `+layout.svelte` file:

```svelte
<script lang="ts">
import { PUBLIC_HUMANBEHAVIOR_API_KEY } from '$env/static/public';
import { browser } from '$app/environment';
import { humanBehaviorStore } from 'humanbehavior-js/svelte';

// Only initialize on client side to avoid SSR issues
if (browser) {
  humanBehaviorStore.init(PUBLIC_HUMANBEHAVIOR_API_KEY || '');
}

let { children } = $props();
</script>

{@render children?.()}
```

<Note>
  **Important:** This setup is for SvelteKit (which has SSR). The `browser` check prevents the tracker from initializing during server-side rendering, which would cause errors since browser APIs aren't available on the server.
</Note>

### For Regular Svelte (non-SSR)

In your `main.js` or `main.ts` file:

```javascript
import App from './App.svelte';
import { humanBehaviorStore } from 'humanbehavior-js/svelte';

// Direct initialization (no browser check needed)
const tracker = humanBehaviorStore.init('your-api-key-here');

const app = new App({
  target: document.body
});

export default app;
```

<Note>
  **That's it!** The SDK now automatically tracks user interactions, navigation, and console events. Everything past here is optional.
</Note>

## Troubleshooting

- Check browser console for any error messages
- Verify your API key is correct
- Ensure the tracker is initialized before any user interaction
- For SvelteKit: Make sure you're using the `browser` check to avoid SSR issues
